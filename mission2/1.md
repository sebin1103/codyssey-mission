# 텍스트 파일과 이진 파일의 차이점 및 장단점

## 1. 개요

* **텍스트 파일**
  * 사람이 직접 읽고 이해할 수 있는 문자 형태로 데이터가 인코딩되어 저장되는 파일
  * 대표적인 확장자: `.txt`, `.csv`, `.html`, `.json` 등

* **이진 파일**
  * 컴퓨터가 직접 처리할 수 있도록 0과 1의 바이너리 형태로 데이터가 있는 그대로 저장되는 파일
  * 특정 프로그램이나 규약이 있어야만 내용을 해석
  * 대표적인 확장자: `.bin`, `.png`, `.exe`, `.mp4` 등

---

## 2. 장단점 비교

| 구분 | 장점 | 단점 |
| :--- | :--- | :--- |
| **텍스트 파일<br>** | - **가독성:** 텍스트 편집기로 열어서 사람이 직접 내용을 확인하고 수정하기 쉬움<br>- **호환성:** 서로 다른 운영체제나 플랫폼 간에 데이터를 공유하고 이식하기가 매우 수월 | - **공간 낭비:** 숫자 데이터 등을 저장할 때 문자열로 변환하여 저장하므로 이진 파일에 비해 용량이 커질 수 있음<br>- **처리 속도 저하:** 컴퓨터가 처리하기 위해 문자를 다시 데이터 형식으로 파싱해야 하므로 속도가 상대적으로 느림|
| **이진 파일<br>** | - **효율성:** 데이터를 변환 없이 바이너리 형태로 저장하므로 저장 공간을 적게 차지<br>- **처리 속도:** 텍스트 인코딩/디코딩 과정이 생략되거나 적어 컴퓨터가 읽고 쓰는 속도가 매우 빠름 | - **가독성 부족:** 일반 텍스트 편집기로 열면 글자가 깨져 보여 사람이 직접 데이터를 확인하거나 수정할 수 없음<br>- **호환성 제한:** 데이터를 작성한 프로그램의 구조나 규약을 알아야만 올바르게 읽을 수 있음 |

---

## 3. 요약 및 사용처

* **텍스트 파일 사용이 유리한 경우**
  * 사람이 직접 읽고 내용을 수정해야 하는 경우 (예: 설정 파일, 소스 코드)
  * 다양한 환경에서 범용적으로 데이터를 공유해야 하는 경우 (예: CSV, JSON 데이터 송수신)

* **이진 파일 사용이 유리한 경우**
  * 저장 공간 최적화가 필요한 대용량 데이터인 경우 (예: 이미지, 영상, 오디오)
  * 프로그램의 빠른 처리 속도가 요구되는 경우 (예: 대규모 배열 데이터, 구조체, 실행 파일)
