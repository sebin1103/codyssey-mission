
# 🚀 화성 기지 보관함 암호 해독 스크립트 (door_hacking.py)

이 프로젝트는 잠겨있는 ZIP 파일(`emergency_storage_key.zip`)의 6자리 암호(소문자+숫자)를 빠르고 효율적으로 해독하기 위해 작성된 파이썬 스크립트입니다. 단순한 무차별 대입(Brute-force) 방식의 한계를 극복하기 위해 사전(Dictionary) 공격을 결합한 하이브리드 방식을 사용합니다.

## 📝 주요 기능 및 로직 파악

본 스크립트는 `unlock_zip()`이라는 단일 함수 내에서 다음의 4가지 주요 단계를 거쳐 실행됩니다.

### 1. 사전 준비 및 파일 예외 처리
* **대상 파일:** `emergency_storage_key.zip`
* **출력 파일:** `password.txt`
* `zipfile` 모듈을 사용하여 파일을 엽니다. 파일이 존재하지 않거나(`FileNotFoundError`), 손상된 파일(`BadZipFile`)인 경우를 대비한 꼼꼼한 예외 처리가 적용되어 있습니다. 암호 테스트 속도를 높이기 위해 전체 압축을 풀지 않고, 파일 내의 첫 번째 요소(`infolist()[0]`)만 읽기 테스트용으로 사용합니다.

### 2. [Stage 1] 우선 탐색 (하이브리드 사전 공격)
단순 무차별 대입 시 소요되는 엄청난 시간(약 21억 번의 연산)을 단축하기 위한 알고리즘입니다. 화성 기지 대원들이 사용했을 법한 유력한 키워드를 바탕으로 약 900개의 조합을 자동 생성하여 먼저 검사합니다.
* **사용된 패턴:**
  * 완전한 6자리 단어 (예: `coffee`, `oxygen`, `planet`)
  * 4자리 단어 + 2자리 숫자 (예: `mars00` ~ `mars99`, `base00` ~ `base99`)
  * 5자리 단어 + 1자리 숫자 (예: `earth0` ~ `earth9`)
* 이 단계에서 암호를 발견하면 즉시 루프를 종료하고 결과를 반환하여 압도적으로 빠른 속도를 보장합니다.

### 3. [Stage 2] 전체 무차별 대입 (Brute-force)
1단계 우선 탐색에서 암호를 찾지 못한 경우, `itertools.product`를 사용하여 소문자 알파벳과 숫자로 구성 가능한 모든 6자리 조합을 순차적으로 대입합니다.
* **진행 상황 모니터링:** I/O 부하를 줄이기 위해 100,000번 시도할 때마다 현재 대입 중인 문자열과 경과 시간을 터미널에 출력합니다.
* **오류 무시(Pass):** 틀린 암호를 입력했을 때 발생하는 파이썬 내부의 압축 해제 에러(`zlib.error`, `RuntimeError`)를 예외 처리 구문으로 잡아내어(`pass`), 프로그램이 중단되지 않고 다음 암호를 계속 테스트하도록 설계되었습니다.

### 4. 결과 출력 및 저장
* 암호 해독에 성공하면 터미널에 성공 메시지, 총 반복 횟수, 소요 시간을 출력합니다.
* 해독된 비밀번호는 `password.txt` 파일에 안전하게 자동 저장됩니다. (저장 실패 시 `IOError` 예외 처리)

## ⚙️ 실행 환경 및 방법

* **요구 사항:** Python 3.x (외부 라이브러리 설치 불필요. 표준 라이브러리만 사용)
* **실행 방법:** 터미널 또는 명령 프롬프트에서 아래 명령어를 입력하여 실행합니다.

```bash
# 대상 zip 파일이 스크립트와 동일한 디렉토리에 있어야 합니다.
python door_hacking.py
